== 发现

发现模块负责发现集群内部的节点，以及选举主节点。

注意，Elasticsearch 是一个基于对等的系统，如果操作被委托/广播，节点之间直接通信。所有的主要 API (索引，删除，查询）都不与主节点通信。主节点的职责是维护全局的集群状态，以及在节点加入或离开集群时进行分片分配。每次集群状态改变时，都会被集群内其他节点获知（方式取决于实际的发现机制的实现）。

[float]
=== 设定

`cluster.name` 可以用来创建彼此分隔的集群。集群名的默认值是 `elasticsearch`，但建议更改该值以反映运行的集群的逻辑组名称。
